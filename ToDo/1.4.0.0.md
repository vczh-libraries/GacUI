# 1.4.0.0

## New Feature

- VlppParser2 generates `.d.ts` schema along with JSON visitor from AST definition.
- Workflow RPC Interfaces and Code Generation.

## Fixed Bugs

- When calling `Event<T>::operator()` with rvalue, the same rvalue argument would be moved multiple times into all handlers.

## Milestone

### Workflow/VlppParser2 AST

- JSON and XML builder API supports looping.
- JSON and XML (or general AST) builder expression, compiled to builder API.
- Or more generally object constructor supports adding members to readonly property returning a mutable container.
- Optimize Workflow RPC JSON Serialization code.

### VlppChannel

- A channel framework focusing on local multi-process information interchange.
  - Windows
    - stdio 
    - Named Pipe
    - HTTP
  - Linux/macOS
    - stdio
    - Socket
  - JSON <-> UTF
  - Copy from GacUI
  - Keep `Submit` but no `ProcessEvents` and `ExecutablePath`.

## Copilot

- `kb-refresh.prompt.md` to find out if any knowledge base needs to update.
- Common monorepo jobs in `Tools`:
  - `MonoRepo.publish.prompt.md` to build, test, download and update `vczh-libraries.github.io`, push to master branch.
  - `MonoRepo.release.prompt.md` to run `Build.ps1` against `Vlpp`, `VlppOS`, `VlppRegex`, `VlppReflection`, `VlppParser2`, `Workflow`, `GacUI`, `Release` in order, fix any issues, push to master branch.

  - `refine.prompt.md`: `copilotRemember.ps1`.
  - `kb-sync.prompt.md`: `copilotInit.ps1`, `copilotInitAll.ps1`.
