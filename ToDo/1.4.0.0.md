### SyncObj

- SyncObj architecture that streams ViewModel object changes.
  - See README.md in Workflow repo (**Workflow Interface Based RPC**).
  - Workflow compiler will output metadata so that other language can do codegen.
  - Network protocols are not included as default implementation
- SyncObj with Remote Protocol.
  - `Protocols.txt` still works for renderer only.
  - Introduces channel name to the underlying channel layer.
    - A client could handle both rendering and SyncObj. The connection should offer both information at once in this case.
  - Multiple clients can connect to GacUI core in the channel layer.
    - Here to solve the `ControllerConnect` problem.
    - Client will no longer send `ControllerConnect` directly, instead two different connect methods will be used by a client to tell if it is a renderer or ordinary client.
      - By if `event->OnControllerConnect` is called, it will be intercepted and re-explained by the channel. So renderer code does not change.
        - `GacJS` will need to change due to test purpose JSON protocol changing.
      - No reconnection allowed in SyncObj, SyncObj disconnection causes GacUI to stop with a fetal error.
      - Distinguish intended disconnection from unexpected disconnection.
    - The `ControllerConnect` signal no longer needs to be parsed from raw JSON messages.
    - When the channel receives a renderer connection, `ControllerConnect` is called by the channel itself.
  - When the renderer connection is received, if GacUI is not running with remote protocol, it will be rejected.
  - When the SyncObj connection is received, if GacUI is not running with SyncObj, it will be rejected.
